**Goal**: `npx @llm-corpus/cli init` on a clean Linux or macOS machine with Node ≥ 18 and Ollama running provisions a working corpus within a 90-second AbortController-bounded budget: preflight → idempotency check → XDG subtree → single-file SQLite → default `config.toml` → curated seed → MCP-client config → OS firewall → optional auto-start → install-receipt → next-step output (steps 1-11). Optional `--smoke` (step 12) runs the C-046 end-to-end harness with its own 30-second sub-budget.

**Independent Test**: With `CORPUS_HOME=<tempdir>` against a clean HOME on pai-node01 (or a clean macOS VM), spawn `node <dist>/bin/corpus.js init`; assert XDG subtree created, SQLite single-file at `Paths.indexDb()`, `config.toml` Zod-validates, ≥ 25 taxonomy_terms rows with `state='established'`, `mcpServers.corpus` entry in `~/.claude.json`, OS firewall rule present (post-uninstall verifiable), install-receipt Zod-validates, exit 0, wall-clock ≤ 90 s. With `--smoke` flag, spawn the production binary, drop the seed doc, observe `edges-build.completed` telemetry, spawn a real `corpus mcp` child, invoke `corpus.find` via real MCP-stdio, assert ≥ 1 SearchHit pointing at the seed.
